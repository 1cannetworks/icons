# Github Pages Image 호스팅 가이드
## 1. 업로드 경로 이동
### 1.1 업로드할 경로가 없는 경우
1. `Add files` $\rightarrow$ `Create new file`  선택 
2. 상단 입력창 `경로명/파일명` 입력
3. `Commit changes...` 클릭
	- 필요시 `Commit message` 또는 `Extended descriptoon` 작성
	- `Commit directly to the main branch` 선택
4. `Commit changes` 클릭
---
## 2. 이미지 업로드
1. `Add files` $\rightarrow$ `Upload files` 클릭
	- 이동한 경로에 파일이 업로드
2. 이미지를 `드래그` 또는 `choose your files` 클릭하여 업로드
	- 필요시 `Commit message` 작성
	- `Commit directly to the main branch` 선택
3. `Commit changes` 클릭
---
## 3. 이미지 URL 추출
1. 이미지를 업로드한 경로에 이동
2. 업로드 완료한 이미지 확인
3. 이미지 경로 복사
	- 업로드 이미지 마우스 우클릭
	- 이미지 주소 복사
---
## 4. 주의사항
### 4.1 파일명 한글, 공백, 특수문자 사용 금지

>영문, 숫자, 하이픈, 언더스코어만 사용

---
### 4.2 이미지 용량은 가급적 1MB 이하로 최적화

>무료 호스팅 가능한 repository는 `1GB` / `월 100GB 트래픽`
>단일 업로드 가능한 파일의 크기는 `100MB` 제한


### 4.3 이미지를 포함한 상세 가이드

> `원캔 게시판` $\rightarrow\$ `개발 공통 사항` $\rightarrow\$ `Github Pages Image 호스팅 가이드` 참고
